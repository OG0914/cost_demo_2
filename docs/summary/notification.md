# 通知模块

## 一、模块概述

### 1.1 定位和作用

通知中心模块用于向用户推送系统消息和业务提醒，包括审核结果通知、价格变动提醒、系统公告等。通过通知中心，用户可以及时了解与自己相关的业务动态。

### 1.2 功能范围

- 通知列表展示
- 未读通知提醒
- 通知筛选
- 标记已读
- 忽略通知
- 通知跳转

### 1.3 与其他模块的关联

| 关联模块 | 关联关系 | 说明 |
|----------|----------|------|
| 审核流程 | 通知触发 | 审核通过/退回产生通知 |
| 基础数据 | 通知触发 | 原料价格变动产生通知 |
| 仪表盘 | 通知展示 | 仪表盘显示最近通知 |
| 成本分析 | 通知跳转 | 点击通知跳转到相关成本分析 |

## 二、通知类型

### 2.1 系统通知

- **内容**：系统公告、维护通知等
- **图标**：蓝色信息图标
- **优先级**：普通

### 2.2 审核相关通知

| 子类型 | 触发条件 | 图标颜色 |
|--------|----------|----------|
| 待审核提醒 | 有新的成本分析提交审核 | 橙色 |
| 审核通过 | 成本分析审核通过 | 绿色 |
| 审核退回 | 成本分析被退回 | 红色 |

### 2.3 价格变动通知

- **触发条件**：原料价格发生变化
- **内容**：变动的原料名称、型号、新旧价格
- **图标**：黄色警告图标
- **优先级**：高

## 三、通知中心页面

### 3.1 页面定位

通知中心页面展示所有通知的列表，支持筛选、标记已读、忽略等操作。

### 3.2 初始化逻辑

1. 加载通知列表数据
2. 加载未读数量
3. 启动定时刷新（每分钟）

### 3.3 功能详细描述

#### 3.3.1 筛选功能

**筛选选项**：
- 全部
- 未读
- 价格变动

**交互**：切换筛选条件后立即刷新列表

#### 3.3.2 通知列表

**列表项内容**：
- 类型图标（左侧）
- 标题
- 内容摘要
- 时间
- 关联标签（型号、物料等）
- 未读标记
- 操作按钮（标记已读、忽略）

**未读样式**：
- 背景色不同（浅蓝色）
- 标题颜色不同（蓝色）
- 显示"未读"标签

#### 3.3.3 标记已读

- **单条标记**：点击通知项或"标记已读"按钮
- **全部标记**：点击"全部已读"按钮
- **确认**：全部标记时需要二次确认

#### 3.3.4 忽略通知

- **操作**：点击关闭/忽略按钮
- **结果**：通知从列表中移除

#### 3.3.5 通知跳转

点击通知项根据类型跳转到对应页面：

| 通知类型 | 跳转目标 |
|----------|----------|
| 成本分析相关 | 成本分析详情页 |
| 标准成本相关 | 标准成本列表 |
| 原料相关 | 原料管理页 |
| 型号相关 | 型号管理页 |

### 3.4 界面设计描述

#### 3.4.1 整体布局

```
┌─────────────────────────────────────────┐
│ 通知中心                    [全部已读]   │
├─────────────────────────────────────────┤
│ [全部] [未读] [价格变动]      [刷新]     │
├─────────────────────────────────────────┤
│                                         │
│  通知列表                                │
│  ┌─────────────────────────────────┐   │
│  │ [图标] 标题              [未读]  │   │
│  │        内容摘要                  │   │
│  │        时间    [型号] [物料]     │   │
│  └─────────────────────────────────┘   │
│                                         │
├─────────────────────────────────────────┤
│ 分页组件                                 │
└─────────────────────────────────────────┘
```

#### 3.4.2 头部设计

- **标题**："通知中心"
- **未读数量**：标题旁显示未读数量徽章（红色）
- **全部已读按钮**：有未读时显示

#### 3.4.3 筛选栏设计

- **单选按钮组**：全部/未读/价格变动
- **刷新按钮**：右侧刷新图标按钮

#### 3.4.4 通知项设计

- **布局**：横向布局，图标+内容+操作
- **图标区域**：40px圆形背景，不同类型不同颜色
- **内容区域**：标题+内容+时间+标签
- **操作区域**：标记已读按钮+忽略按钮（悬停显示）

### 3.5 按钮设计描述

| 按钮 | 样式 | 位置 | 交互 |
|------|------|------|------|
| 全部已读 | 主按钮 | 头部右侧 | 确认后标记所有为已读 |
| 刷新 | 图标按钮 | 筛选栏右侧 | 点击刷新列表 |
| 标记已读 | 文字按钮（蓝色） | 通知项右侧 | 点击标记该条为已读 |
| 忽略 | 图标按钮 | 通知项右侧 | 点击移除该通知 |

## 四、通知展示规则

### 4.1 未读通知角标

- **位置**：导航栏通知图标右上角
- **样式**：红色圆形徽章
- **显示条件**：有未读通知时显示
- **数字**：显示未读数量，超过99显示"99+"

### 4.2 通知排序

- **默认排序**：按时间倒序（最新的在前）
- **未读优先**：未读通知排在已读之前

### 4.3 通知保留期限

- **系统默认**：通知保留30天
- **自动清理**：超过期限的通知自动删除
- **重要通知**：审核相关通知保留90天

## 五、状态与提示

### 5.1 页面状态

| 状态 | 视觉表现 | 说明 |
|------|----------|------|
| 加载中 | 列表loading | 首次加载数据 |
| 空数据 | 空状态插图 | 无通知时显示 |
| 正常状态 | 通知列表 | 有通知时显示 |

### 5.2 空状态设计

- **图标**：收件箱图标
- **标题**："暂无通知"
- **说明**："当物料价格变动时，您将收到相关提醒"

### 5.3 操作反馈

| 操作 | 成功提示 | 失败提示 |
|------|----------|----------|
| 标记已读 | 无 | "标记已读失败" |
| 全部已读 | "已全部标记为已读" | "标记全部已读失败" |
| 忽略通知 | "通知已关闭" | "关闭通知失败" |

## 六、自动刷新机制

### 6.1 定时刷新

- **间隔**：60秒
- **触发**：自动获取最新通知
- **更新**：有新通知时更新列表和角标

### 6.2 手动刷新

- **入口**：筛选栏刷新按钮
- **交互**：点击立即刷新
- **反馈**：刷新完成后提示

## 七、通知生成规则

### 7.1 审核通知

| 场景 | 接收人 | 内容 |
|------|--------|------|
| 成本分析提交 | 审核员 | "有新的成本分析待审核" |
| 审核通过 | 创建人 | "您的成本分析已通过审核" |
| 审核退回 | 创建人 | "您的成本分析被退回，原因：xxx" |

### 7.2 价格变动通知

| 场景 | 接收人 | 内容 |
|------|--------|------|
| 原料价格上涨 | 相关用户 | "原料xxx价格上涨，从x元涨至x元" |
| 原料价格下跌 | 相关用户 | "原料xxx价格下跌，从x元降至x元" |
