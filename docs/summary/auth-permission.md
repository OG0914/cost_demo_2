# 认证与权限模块

## 一、模块概述

### 1.1 定位和作用

认证与权限模块是系统的入口和安全守门人，负责用户身份验证、权限控制和角色管理。该模块确保只有授权用户才能访问系统功能，并根据用户角色限制可操作的数据范围。

### 1.2 功能范围

- 用户登录认证
- 权限检查与控制
- 角色管理
- 密码修改

### 1.3 与其他模块的关联

| 关联模块 | 关联关系 | 说明 |
|----------|----------|------|
| 所有模块 | 依赖 | 所有功能都需要先通过认证 |
| 系统配置 | 被依赖 | 角色权限配置影响系统功能可见性 |
| 个人中心 | 关联 | 密码修改功能在个人中心实现 |

## 二、登录页面

### 2.1 页面定位

登录页面是系统的入口页面，未登录用户访问任何功能都会被重定向到此页面。

### 2.2 初始化逻辑

1. 页面加载时检查本地是否存储了登录凭证
2. 若已登录且凭证有效，自动跳转到仪表盘
3. 显示登录表单，包含账号和密码输入框

### 2.3 功能详细描述

#### 2.3.1 账号输入

- **输入框类型**：文本输入框
- **占位提示**："请输入用户名"
- **验证规则**：必填，长度2-50字符
- **自动完成**：支持浏览器自动填充

#### 2.3.2 密码输入

- **输入框类型**：密码输入框（可切换显示/隐藏）
- **占位提示**："请输入密码"
- **验证规则**：必填，最小6位
- **密码显示切换**：点击眼睛图标可在明文/密文间切换

#### 2.3.3 登录按钮

- **按钮样式**：主按钮（蓝色填充），圆角设计
- **按钮位置**：表单底部居中
- **交互反馈**：
  - 点击后显示加载动画
  - 登录中按钮文字变为"登录中..."
  - 登录成功跳转至仪表盘
  - 登录失败显示错误提示

### 2.4 界面设计描述

#### 2.4.1 整体布局

采用左右分栏设计：
- **左侧（大屏显示）**：品牌展示区，蓝色渐变背景，包含系统Logo、名称、标语
- **右侧（全屏显示）**：登录表单区，白色背景

#### 2.4.2 品牌展示区

- **背景**：蓝色渐变（#2563eb 到 #60a5fa）
- **Logo**：系统Logo图片，白色滤镜
- **系统名称**："成本分析系统"，白色大字
- **标语**："数据驱动 · 精准决策 · 高效管理"
- **底部说明**："内部专用 · 数据保密 · 高效决策"

#### 2.4.3 表单区

- **标题**："欢迎回来"
- **副标题**："登录您的账户以继续"
- **输入框样式**：圆角边框，聚焦时蓝色边框
- **移动端适配**：小屏设备隐藏左侧品牌区，表单居中显示

### 2.5 状态与提示

| 状态 | 提示方式 | 说明 |
|------|----------|------|
| 加载中 | 按钮loading动画 | 登录请求处理中 |
| 登录成功 | 跳转页面 | 自动跳转到仪表盘 |
| 账号错误 | 错误提示 | "用户名或密码错误" |
| 网络错误 | 错误提示 | "网络连接失败，请重试" |
| 账号禁用 | 错误提示 | "账号已被禁用，请联系管理员" |

## 三、权限控制

### 3.1 权限体系

系统采用基于角色的访问控制（RBAC）模型，权限分为两个层级：

#### 3.1.1 角色层级

| 角色 | 代码 | 权限范围 |
|------|------|----------|
| 管理员 | admin | 所有功能 |
| 审核员 | reviewer | 审核相关功能 |
| 业务员 | salesperson | 成本分析相关功能 |
| 采购员 | purchaser | 原料、包材管理 |
| 生产人员 | producer | 工序、型号管理 |
| 只读用户 | readonly | 仅查看权限 |

#### 3.1.2 权限码定义

| 权限码 | 说明 | 所属角色 |
|--------|------|----------|
| cost:create | 创建成本分析 | 业务员、管理员 |
| cost:edit | 编辑成本分析 | 业务员(自己的)、管理员 |
| cost:delete | 删除成本分析 | 业务员(自己的)、管理员 |
| cost:delete:all | 删除任意成本分析 | 管理员 |
| cost:view | 查看成本分析 | 所有角色 |
| cost:manage | 管理标准成本 | 审核员、管理员 |
| review:view | 查看审核记录 | 审核员、管理员、业务员(自己的) |
| review:approve | 执行审核操作 | 审核员、管理员 |
| master:material:view | 查看原料 | 采购员、管理员 |
| master:material:manage | 管理原料 | 采购员、管理员 |
| master:packaging:view | 查看包材 | 采购员、管理员 |
| master:packaging:manage | 管理包材 | 采购员、管理员 |
| master:process:view | 查看工序 | 生产人员、管理员 |
| master:process:manage | 管理工序 | 生产人员、管理员 |
| master:model:view | 查看型号 | 生产人员、管理员 |
| master:model:manage | 管理型号 | 生产人员、管理员 |
| master:customer:view | 查看客户 | 业务员、管理员 |
| master:customer:manage | 管理客户 | 业务员、管理员 |
| master:regulation:view | 查看法规 | 所有角色 |
| master:regulation:manage | 管理法規 | 管理员 |
| system:admin | 系统管理权限 | 管理员 |
| system:config:manage | 系统配置管理 | 管理员 |

### 3.2 权限检查逻辑

#### 3.2.1 前端权限检查

- **实现方式**：通过 `useAuthStore` 提供的 `hasPermission` 方法检查
- **应用场景**：
  - 控制按钮/菜单的显示/隐藏
  - 控制页面访问权限
  - 控制表格操作列的显示

#### 3.2.2 后端权限检查

- **实现方式**：通过JWT Token验证用户身份，中间件检查权限
- **应用场景**：
  - API接口访问控制
  - 数据范围控制（业务员只能看自己的数据）
  - 敏感操作二次验证

### 3.3 数据范围控制

#### 3.3.1 业务员数据隔离

业务员角色存在特殊的数据范围限制：

| 功能 | 数据范围 |
|------|----------|
| 成本记录 | 只能查看自己创建的成本分析 |
| 待审核列表 | 只能查看自己提交的成本分析 |
| 已审核列表 | 只能查看自己提交的成本分析 |
| 草稿 | 只能查看自己的草稿 |

#### 3.3.2 审核员数据范围

审核员可以查看所有待审核和已审核的记录，但不能查看他人的草稿。

## 四、角色管理

### 4.1 页面定位

角色管理页面用于管理系统中的角色定义，包括角色的增删改查和状态管理。仅管理员可访问此页面。

### 4.2 初始化逻辑

1. 页面加载时获取角色列表
2. 并行获取各角色的权限数量
3. 渲染角色列表表格

### 4.3 功能详细描述

#### 4.3.1 角色列表

- **展示形式**：表格
- **表格列**：
  - 序号
  - 角色信息（图标、名称、代码、系统标签）
  - 描述
  - 权限数（可点击跳转权限配置）
  - 状态（启用/禁用开关）
  - 操作（编辑、权限、删除）

#### 4.3.2 新增角色

- **入口**：页面右上角"新增角色"按钮
- **表单字段**：
  - 角色名称（必填）
  - 角色代码（必填，唯一）
  - 描述（选填）
  - 图标（选填）
- **验证规则**：
  - 角色代码只能包含小写字母和下划线
  - 角色代码不能重复

#### 4.3.3 编辑角色

- **入口**：表格行内"编辑"按钮
- **可编辑字段**：角色名称、描述、图标
- **限制**：系统内置角色（admin等）部分字段不可编辑

#### 4.3.4 删除角色

- **入口**：表格行内"删除"按钮
- **限制**：系统内置角色不可删除
- **确认**：删除前需要二次确认
- **影响**：删除角色后，该角色下的用户将失去相应权限

#### 4.3.5 角色状态切换

- **控制方式**：表格行内开关组件
- **功能**：启用/禁用角色
- **限制**：系统内置角色不可禁用

### 4.4 界面设计描述

#### 4.4.1 整体布局

- **顶部**：页面标题"角色管理" + 描述文字 + 新增按钮
- **主体**：角色列表表格

#### 4.4.2 角色信息展示

每个角色以卡片式信息展示：
- **图标**：角色对应的大图标，不同角色不同颜色
- **名称**：角色显示名称
- **代码**：角色唯一标识代码
- **标签**：系统角色显示"系统"标签，禁用角色显示"禁用"标签

### 4.5 按钮设计描述

| 按钮 | 样式 | 位置 | 权限要求 |
|------|------|------|----------|
| 新增角色 | 主按钮 | 页面右上角 | system:admin |
| 编辑 | 文字按钮（蓝色） | 表格行内操作列 | system:admin |
| 权限 | 文字按钮（蓝色） | 表格行内操作列 | system:admin |
| 删除 | 文字按钮（红色） | 表格行内操作列 | system:admin |
| 状态开关 | 开关组件 | 表格行内状态列 | system:admin |

## 五、密码修改

### 5.1 功能定位

密码修改功能位于个人中心页面，允许用户修改自己的登录密码。

### 5.2 功能详细描述

#### 5.2.1 旧密码输入

- **输入类型**：密码输入框
- **验证规则**：必填，与当前密码匹配
- **错误提示**："旧密码不正确"

#### 5.2.2 新密码输入

- **输入类型**：密码输入框
- **验证规则**：
  - 必填
  - 最小6位
  - 不能与旧密码相同
- **强度提示**：根据密码复杂度显示强度等级

#### 5.2.3 确认密码输入

- **输入类型**：密码输入框
- **验证规则**：必填，必须与新密码一致
- **错误提示**："两次输入的密码不一致"

#### 5.2.4 确认按钮

- **按钮样式**：主按钮
- **交互反馈**：
  - 点击后验证表单
  - 验证通过后显示loading状态
  - 修改成功后提示"密码修改成功"
  - 自动清空表单

### 5.3 状态与提示

| 状态 | 提示方式 | 说明 |
|------|----------|------|
| 表单验证失败 | 字段级错误提示 | 显示在对应输入框下方 |
| 旧密码错误 | 全局错误提示 | "旧密码不正确" |
| 修改成功 | 成功提示 | "密码修改成功" |
| 网络错误 | 错误提示 | "网络连接失败，请重试" |

## 六、安全机制

### 6.1 认证机制

- **认证方式**：JWT Token
- **Token存储**：localStorage
- **Token有效期**：24小时
- **自动续期**：每次请求自动刷新Token

### 6.2 会话管理

- **单点登录**：同一账号同一时间只能在一个设备登录
- **超时处理**：Token过期后自动跳转登录页
- **退出登录**：清除本地存储的Token和用户信息

### 6.3 密码安全

- **存储方式**：后端使用BCrypt加密存储
- **传输安全**：HTTPS传输
- **密码策略**：最小6位，建议包含字母和数字
