# 成本分析管理系统 - UI设计规范

> 本文档用于在新架构（Next.js + React）上复刻前端页面样式
> 提取自原项目（Vue 3 + Element Plus + Tailwind CSS）

---

## 一、设计系统概述

### 1.1 设计理念

- **简洁专业**：以功能为导向，减少视觉干扰
- **层次分明**：通过色彩、间距建立清晰的视觉层级
- **响应式适配**：支持桌面端和移动端

### 1.2 技术栈对应关系

| 原项目 | 新项目 | 对应方案 |
|-------|-------|---------|
| Element Plus | shadcn/ui 或 Ant Design | 推荐 shadcn/ui，可高度定制 |
| Tailwind CSS | Tailwind CSS | 配置相同，直接复用 |
| Vue 3 | React 19 | 组件逻辑重写 |

## 十三、动画规范

### 13.1 过渡动画

| 动画 | 时长 | 缓动函数 | 用途 |

| **弹窗出现** | 300ms | ease-out | 弹窗打开 |
| **菜单展开** | 200ms | ease-in-out | 子菜单展开 |
| **列表项** | 150ms | ease | 列表项Hover |

### 13.2 页面淡入动画

```css
@keyframes fadeIn {
  from {
    opacity: 0;
    transform: translateY(10px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}
```

| **右侧面板** | 隐藏，改为底部固定栏或抽屉 | 点击"预览"按钮展开 |
| **表格** | 横向滚动或卡片化 | 优先使用卡片列表 |
| **表单** | 单列布局 | 标签在上，输入框在下 |
| **侧边栏** | 收起为抽屉菜单 | 点击汉堡菜单展开 |
| **操作按钮** | 底部固定栏 | 重要操作始终可见 |
| **筛选区** | 可折叠面板 | 默认收起，节省空间 |

### 17.3 校验时机

| 校验类型 | 触发时机 | 说明 |
|---------|---------|------|
| 必填校验 | 失焦时（blur） | 离开输入框时检查 |
| 格式校验 | 输入时（input） | 实时检查格式 |
| 长度校验 | 失焦时（blur） | 离开输入框时检查 |
| 范围校验 | 失焦时（blur） | 离开输入框时检查 |
| 提交校验 | 点击提交按钮 | 提交前全量校验 |

| 禁用 | Slate-200 | Slate-100 | - |

### 17.5 校验提示信息

**提示信息位置**：
- 单行表单：输入框下方
- 多行表单：输入框右侧或下方
- 弹窗表单：统一在底部显示

**提示信息格式**：
```
┌─────────────────────────────────┐
│ [输入框                        ]│
│ ⚠ 错误提示信息                  │  ← 图标+文字
└─────────────────────────────────┘
```

**常见校验提示**：

| 校验规则 | 提示信息 |
|---------|---------|
| 必填 | "请输入{字段名}" |
| 最小长度 | "{字段名}至少需要{长度}个字符" |
| 最大长度 | "{字段名}不能超过{长度}个字符" |
| 数字范围 | "{字段名}必须在{最小值}到{最大值}之间" |
| 邮箱格式 | "请输入有效的邮箱地址" |
| 手机号格式 | "请输入有效的手机号码" |
| 唯一性 | "{字段名}已存在，请更换" |
