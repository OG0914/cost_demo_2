# 个人中心模块

## 一、模块概述

### 1.1 定位和作用

个人中心模块用于展示当前登录用户的个人信息，提供密码修改等功能。用户可以在此查看自己的账号信息、角色权限，并修改登录密码。

### 1.2 功能范围

- 个人信息展示
- 密码修改

### 1.3 与其他模块的关联

| 关联模块 | 关联关系 | 说明 |
|----------|----------|------|
| 认证与权限 | 数据关联 | 显示用户认证信息 |
| 系统配置 | 权限关联 | 根据角色显示不同信息 |

## 三、功能详细描述

### 3.1 个人信息展示

#### 3.1.1 信息卡片

**卡片内容**：

| 信息项 | 说明 | 展示方式 |
|--------|------|----------|
| 头像 | 用户头像 | 圆形头像，显示姓名首字母 |
| 用户名 | 登录账号 | 文字显示 |
| 真实姓名 | 用户真实姓名 | 文字显示，大号字体 |
| 角色 | 用户角色 | 彩色标签 |
| 邮箱 | 用户邮箱 | 图标+文字 |
| 状态 | 账号状态 | 状态指示器+文字 |

#### 3.1.2 头像设计

- **形状**：圆形
- **尺寸**：56px
- **背景色**：根据角色显示不同颜色
- **文字**：真实姓名的首字母（大写）
- **颜色**：白色文字

#### 3.1.3 角色标签

**角色颜色映射**：

| 角色 | 代码 | 颜色 |
|------|------|------|
| 管理员 | admin | 红色 (#F56C6C) |
| 采购 | purchaser | 橙色 (#E6A23C) |
| 生产 | producer | 绿色 (#67C23A) |
| 审核 | reviewer | 蓝色 (#409EFF) |
| 业务 | salesperson | 紫色 (#9B59B6) |
| 只读 | readonly | 灰色 (#909399) |

**标签样式**：
- 尺寸：small
- 效果：深色主题（effect="dark"）
- 无边框

#### 3.1.4 状态指示器

- **形状**：圆形小点
- **颜色**：绿色（已启用）
- **文字**："已启用"
- **动画**：脉冲动画效果

### 3.2 密码修改

#### 3.2.1 表单字段

| 字段 | 类型 | 必填 | 验证规则 |
|------|------|------|----------|
| 旧密码 | 密码输入框 | 是 | 必填，与当前密码匹配 |
| 新密码 | 密码输入框 | 是 | 必填，最少6位 |
| 确认密码 | 密码输入框 | 是 | 必填，必须与新密码一致 |

#### 3.2.2 表单验证

**旧密码验证**：
- 必填验证
- 提交后后端验证是否正确
- 错误提示："旧密码不正确"

**新密码验证**：
- 必填验证
- 最小长度6位
- 不能与旧密码相同
- 错误提示："密码长度至少6位"

**确认密码验证**：
- 必填验证
- 必须与新密码一致
- 错误提示："两次输入的密码不一致"

#### 3.2.3 密码强度

- **显示位置**：新密码输入框下方
- **强度等级**：弱/中/强
- **判断依据**：
  - 弱：纯数字或纯字母，少于8位
  - 中：数字+字母，8位以上
  - 强：数字+字母+特殊字符，8位以上

#### 3.2.4 提交按钮

- **样式**：主按钮（蓝色）
- **文字**："确认"
- **加载状态**：提交时显示loading
- **成功反馈**："密码修改成功"
- **清空表单**：成功后清空所有输入

## 四、界面设计描述

### 4.1 整体样式

- **背景**：浅灰色背景
- **卡片**：白色背景，圆角，边框
- **间距**：卡片间距24px

### 4.2 个人信息卡片

#### 4.2.1 卡片结构

```
┌─────────────────────────┐
│                         │
│      ┌─────────┐        │
│      │   头像   │        │
│      └─────────┘        │
│                         │
│       用户名            │
│       真实姓名          │
│       [角色标签]        │
│                         │
│  ─────────────────────  │
│                         │
│  📧 邮箱                │
│  ● 已启用               │
│                         │
└─────────────────────────┘
```

#### 4.2.2 用户头部区域

- **布局**：flex布局，垂直居中
- **头像**：56px圆形
- **间距**：头像与文字间距12px
- **文字间距**：用户名与真实姓名间距4px

#### 4.2.3 用户详情区域

- **布局**：垂直排列
- **图标**：邮箱图标、状态指示器
- **间距**：每项间距50px（设计如此）

### 4.3 修改密码区域

#### 4.3.1 区域结构

```
┌──────────────────────────────┐
│ 修改密码                      │  ← 区域标题
├──────────────────────────────┤
│                              │
│  旧密码                      │
│  [                    ]      │
│                              │
│  新密码                      │
│  [                    ]      │
│  [密码强度: ●○○ 弱]          │
│                              │
│  确认密码                    │
│  [                    ]      │
│                              │
│  [确认]                      │
│                              │
└──────────────────────────────┘
```

#### 4.3.2 区域标题

- **文字**："修改密码"
- **字体**：15px
- **字重**：600（半粗）
- **下边距**：16px
- **边框**：底部1px边框分隔

#### 4.3.3 表单布局

- **标签宽度**：100px
- **输入框宽度**：自适应
- **标签对齐**：右对齐
- **表单项间距**：22px

#### 4.3.4 输入框样式

- **类型**：密码输入框
- **显示切换**：支持显示/隐藏密码
- **占位提示**：
  - 旧密码："请输入旧密码"
  - 新密码："请输入新密码（至少6位）"
  - 确认密码："请再次输入新密码"

## 五、按钮设计描述

| 按钮 | 样式 | 位置 | 交互 |
|------|------|------|------|
| 确认 | 主按钮 | 密码表单底部 | 点击提交表单 |

## 六、状态与提示

### 6.1 表单状态

| 状态 | 视觉表现 | 说明 |
|------|----------|------|
| 初始状态 | 表单可编辑 | 页面加载完成 |
| 验证失败 | 字段级错误提示 | 输入不符合规则 |
| 提交中 | 按钮loading | 请求处理中 |
| 提交成功 | 成功提示，表单清空 | 密码修改成功 |
| 提交失败 | 错误提示 | 显示具体错误原因 |

### 6.2 错误提示

| 场景 | 提示内容 |
|------|----------|
| 旧密码为空 | "请输入旧密码" |
| 旧密码错误 | "旧密码不正确" |
| 新密码为空 | "请输入新密码" |
| 新密码太短 | "密码长度至少6位" |
| 确认密码为空 | "请确认新密码" |
| 两次密码不一致 | "两次输入的密码不一致" |

### 6.3 成功提示

- **提示内容**："密码修改成功"
- **提示类型**：成功消息（绿色）
- **自动关闭**：3秒后自动关闭

## 七、数据加载

### 7.1 初始化加载

1. 调用 `/auth/me` 接口获取用户信息
2. 填充个人信息卡片
3. 根据角色设置头像颜色

### 7.2 错误处理

- **加载失败**：显示"加载用户信息失败"提示
- **重试**：提示用户刷新页面重试

## 八、安全考虑

### 8.1 密码传输

- **加密方式**：HTTPS传输
- **密码显示**：默认隐藏，可切换显示

### 8.2 会话安全

- **Token验证**：修改密码需要有效Token
- **超时处理**：Token过期需重新登录
