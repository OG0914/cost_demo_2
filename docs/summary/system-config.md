# 系统配置模块

## 一、模块概述

### 1.1 定位和作用

系统配置模块用于管理成本分析计算所需的各种业务参数和规则配置。这些配置直接影响成本计算的结果，包括费率、汇率、系数、运费参数等。只有管理员可以访问和修改系统配置。

### 1.2 功能范围

- 业务配置（费率、汇率、运费等）
- 计算规则配置
- 基础数据配置（原料类别）

### 1.3 与其他模块的关联

| 关联模块 | 关联关系 | 说明 |
|----------|----------|------|
| 成本分析 | 参数依赖 | 成本计算使用系统配置的参数 |
| 基础数据 | 配置关联 | 原料类别配置影响原料管理 |
| 所有模块 | 参数依赖 | 各模块使用配置的系统参数 |

## 三、计算规则配置

### 3.1 功能说明

计算规则配置用于设置半面罩产品不同计算类型的原料和包材计算公式及系数。

### 3.2 计算类型

| 计算类型 | 说明 |
|----------|------|
| 主体 | 半面罩主体部分 |
| 配件 | 各类配件 |
| 滤毒盒 | 滤毒盒组件 |
| 滤棉 | 滤棉组件 |
| 滤饼 | 滤饼组件 |

### 3.3 计算公式

#### 3.3.1 乘法公式

```
小计 = (用量 × 单价) ÷ 系数
```

#### 3.3.2 除法公式

```
小计 = (单价 ÷ 用量) ÷ 系数
```

### 3.4 配置表格

| 计算类型 | 原料公式 | 原料系数 | 包材公式 | 包材系数 |
|----------|----------|----------|----------|----------|
| 主体 | 乘/除 | 数值 | 乘/除 | 数值 |
| 配件 | 乘/除 | 数值 | 乘/除 | 数值 |
| 滤毒盒 | 乘/除 | 数值 | 乘/除 | 数值 |
| 滤棉 | 乘/除 | 数值 | 乘/除 | 数值 |
| 滤饼 | 乘/除 | 数值 | 乘/除 | 数值 |

### 3.5 配置编辑

- **公式选择**：下拉选择"乘"或"除"
- **系数输入**：数字输入框，支持小数
- **批量设置**：可单独设置每个计算类型

### 3.6 保存规则

- **保存按钮**："保存计算规则"按钮
- **验证**：检查公式有效性和系数范围
- **提示**：保存成功后显示提示

## 四、基础数据配置

### 4.1 原料类别管理

#### 4.1.1 功能说明

管理原料的分类，用于原料管理页面的筛选和分组。

#### 4.1.2 类别字段

| 字段 | 说明 |
|------|------|
| 类别名称 | 分类名称，如"活性炭"、"滤材"等 |
| 排序 | 显示顺序，数字越小越靠前 |

#### 4.1.3 操作

- **新增类别**：点击"新增类别"按钮，弹出对话框输入名称和排序
- **编辑类别**：直接在表格中修改
- **删除类别**：点击删除按钮，确认后删除
- **保存配置**：点击"保存类别配置"按钮保存所有修改

#### 4.1.4 验证规则

- 类别名称不能为空
- 类别名称不能重复
- 排序为1-999之间的整数

### 5.1 整体布局

采用Tab切换设计：

```
┌─────────────────────────────────────────┐
│ 页面标题                                │
├─────────────────────────────────────────┤
│ [业务配置] [计算规则配置] [基础数据配置] │
├─────────────────────────────────────────┤
│                                         │
│  Tab内容区域                             │
│                                         │
└─────────────────────────────────────────┘
```

### 5.2 业务配置Tab布局

采用Bento Grid布局：

```
┌─────────┬─────────┬─────────┐
│ 基础费率 │ 汇率与  │ 汇率与  │
│ (1列)   │ 系数    │ 系数    │
│         │ (2列)   │ (续)    │
├─────────┴─────────┴─────────┤
│ CIF深圳运费 (3列)            │
├─────────┬───────────────────┤
│ 整柜运费 │ 散货运费          │
│ (1列)   │ (2列)             │
├─────────┴───────────────────┤
│ 利润档位 (3列)               │
└─────────────────────────────┘
```

### 5.3 卡片设计

- **头部**：图标 + 标题
- **内容**：配置项表单
- **样式**：白色背景，圆角，边框
- **悬停效果**：轻微阴影

### 5.4 输入框设计

- **数字输入**：无控制按钮的数字输入框
- **单位显示**：输入框右侧显示单位
- **百分比输入**：输入小数，自动显示百分比

## 七、状态与提示

### 7.1 未保存提示

- **检测**：监听配置变化
- **提示方式**：
  - 离开页面时弹出确认对话框
  - 浏览器关闭/刷新时提示
- **提示内容**："您有未保存的修改，确定要离开吗？"

### 7.2 验证提示

| 场景 | 提示内容 |
|------|----------|
| 管销率超出范围 | "管销率必须在0到1之间" |
| 汇率无效 | "汇率必须大于0" |
| 利润档位为空 | "至少需要一个利润档位" |
| 类别名称为空 | "请输入类别名称" |
| 类别名称重复 | "该类别已存在" |

### 7.3 操作反馈

| 操作 | 成功提示 | 失败提示 |
|------|----------|----------|
| 保存业务配置 | "配置保存成功" | "保存配置失败" |
| 保存计算规则 | "计算规则保存成功" | "保存计算规则失败" |
| 保存类别配置 | "类别配置保存成功" | "保存类别配置失败" |

## 八、权限控制

- **访问权限**：仅管理员可访问
- **编辑权限**：仅管理员可编辑
- **查看权限**：仅管理员可查看

权限码：`system:config:manage`

## 九、配置生效机制

### 9.1 即时生效

- 配置保存后立即生效
- 前端配置Store自动刷新

### 9.2 影响范围

- **新报价单**：使用最新配置
- **已有报价单**：保持原有计算结果不变
- **草稿**：重新计算时使用最新配置

### 9.3 更新时间显示

- 页面标题右侧显示最后更新时间
- 格式："更新于 YYYY-MM-DD HH:mm"
